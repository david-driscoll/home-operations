# Docker Services Reference

Docker/Dockge stack configurations deployed across all Dockge clusters.

## Directory Structure

```
docker/
├── _common/                    # Shared compose templates & configs
│   ├── authentik-outpost/      # Authentik OAuth2 reverse proxy
│   ├── autoheal/               # Docker container auto-healing
│   ├── backrest/               # Backup orchestration
│   ├── backups/                # Backup services
│   ├── docker-socket-proxy/    # Secure Docker socket proxy
│   ├── dockge/                 # Dockge manager UI
│   ├── librespeed/             # Speed test service
│   ├── neo4j/                  # Graph database
│   ├── openspeedtest/          # Speed test (alternative)
│   ├── prometheus/             # Monitoring & metrics
│   ├── rclone-sftp/            # SFTP backup client
│   ├── technitium/             # Self-hosted DNS server
│   └── traefik/                # Reverse proxy & ingress
├── alpha-site/                 # Alpha Site Dockge cluster
├── celestia/                   # Celestia Dockge cluster
├── luna/                       # Luna Dockge cluster
└── skystar/                    # Skystar Dockge cluster
```

All four cluster directories (`alpha-site/`, `celestia/`, `luna/`, `skystar/`) are **Dockge clusters** — Docker Compose stacks running in LXC containers on Proxmox hosts. None are Kubernetes clusters.

## Common Services (`_common/`)

Shared templates reused or deployed across all clusters.

### Authentik Outpost (`authentik-outpost/`)

OAuth2 proxy for external service authentication. Provides OIDC/OAuth2 integration and user session management. Deployed alongside services that require SSO.

---

### Autoheal (`autoheal/`)

Automatically restarts unhealthy Docker containers. Monitors container health status and applies restart policy. Integrated with notification services.

---

### Backrest (`backrest/`)

Backup orchestration and scheduling. Provides:
- Incremental backups with deduplication
- Schedule and retention management
- Volume backup and restore
- Integration with Rclone SFTP for off-site targets

---

### Backups (`backups/`)

Backup storage and management services — B2 uploader, backup verification scripts, retention policies.

---

### Docker Socket Proxy (`docker-socket-proxy/`)

Secure access layer over the Docker daemon socket. Prevents direct daemon access by exposing only permitted API endpoints.

**Deployment:** Deployed on **ALL Dockge clusters** (Celestia, Luna, Skystar, Alpha Site). Services needing Docker API access must consume this proxy instead of mounting the socket directly.

**Used by:** Dockge manager, Autoheal, container monitoring, Traefik service discovery.

---

### Dockge (`dockge/`)

Web UI for Docker Compose management. Provides real-time container logs, one-click deployments, and compose file editing. Runs as an LXC container on each Proxmox host and is accessible via Tailscale/Traefik.

---

### LibreSpeed (`librespeed/`)

Network speed test service. Measures upload/download speed and latency with historical data and shareable results. Non-critical — safe to test on Alpha Site first.

---

### Neo4j (`neo4j/`)

Graph database for knowledge base and relationship data. Includes APOC procedures and Neo4j Browser UI. Used by llm-graph-builder for knowledge graph queries and network relationship mapping.

---

### OpenSpeedTest (`openspeedtest/`)

Lightweight HTML5-based speed test. Alternative to LibreSpeed with no database requirement.

---

### Prometheus (`prometheus/`)

Metrics collection and monitoring. Components:
- Prometheus server with configurable retention
- Alertmanager integration
- Grafana datasource
- Scrapers: node exporter (system), cAdvisor (containers), custom exporters

---

### Rclone SFTP (`rclone-sftp/`)

SFTP access for remote backups. Presents Rclone-mounted storage as an SFTP target with encryption support. Used for B2 off-site backups and NAS backup access.

---

### Technitium (`technitium/`)

Self-hosted DNS server. Provides DNS filtering, zone management, recursive resolution, and query logging. Configured as authoritative DNS for clusters and integrates with AdGuard for failover.

---

### Traefik (`traefik/`)

Reverse proxy and edge router. Features:
- HTTP/HTTPS routing and TLS termination
- Automatic ACME (Let's Encrypt) certificates via DNS challenge
- Service discovery via Docker labels
- Middleware support (auth, rate limiting, etc.)

**Configuration layout:**
- `traefik.yml` — Static config (backends, logging)
- `dynamic/` folder — Dynamic routing rules (generated by Pulumi at deploy time)
- Per-service Docker labels — Route definition

## Docker Compose Patterns

### Common Service Folder Structure

```
service-name/
├── docker-compose.yml   # Service definition
├── .env (gitignored)    # Environment variables
├── config/              # Configuration files
├── data/ (gitignored)   # Persistent data volume
└── logs/ (gitignored)   # Service logs
```

### Networking

Services connect via Docker networks:

```yaml
networks:
  - traefik      # For services exposed via reverse proxy
  - internal     # Internal services (databases, caches)
  - default      # Default bridge network
```

### Traefik Integration

Services define routes via Docker labels:

```yaml
labels:
  - traefik.enable=true
  - traefik.http.routers.myservice.rule=Host(`myservice.driscoll.tech`)
  - traefik.http.routers.myservice.entrypoints=websecure
  - traefik.http.services.myservice.loadbalancer.server.port=8080
  - traefik.http.routers.myservice.tls.certresolver=letsencrypt
```

### Volume Mounts

```yaml
volumes:
  - data:/app/data          # Named volume (Docker-managed)
  - /mnt/storage:/data      # Host mount
  - ./config:/etc/app       # Relative config mount
```

## Deployment Workflow

### Pulumi-Managed (initial provisioning)

```
Pulumi Stack (DockgeLxc component)
  ↓
Provision LXC Container on Proxmox
  ↓
Install Docker runtime
  ↓
Create Docker Compose stack directory
  ↓
Generate docker-compose.yml from templates
  ↓
Generate .env with credentials from 1Password
  ↓
Run: docker-compose up -d
  ↓
Register with Traefik + update DNS/Tailscale
```

### Manual Updates (after initial deploy)

```bash
cd /path/to/service
docker-compose pull
docker-compose up -d
```

Or use the Dockge UI for non-critical services.
